# 💻 [2026-01] 나만의 C언어 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
이자카야 아르바이트 상황을 기반으로 한 시뮬레이션 시스템

<br>


## 2. 시나리오 (5~10줄)
이 프로그램은 이자카야 술집의 아르바이트생이 되었다고 생각하고 메뉴를 입력해 주문을 받고 최종 계산을 하는 시스템입니다.

프로그램이 시작되면 메뉴판을 출력하고, 사용자에게 주문할 메뉴의 수량을 입력받습니다. 술과 안주 각각의 수량을 입력 받아, 각 항목의 가격을 계산하고 총액에 반영합니다. 총액 출력 전, 할인율을 적용한 후 할인된 총액을 계산하여 출력합니다.
주문 후에는 사용자가 추가 주문을 할지에 대한 여부를 물어보고, 추가 주문이 필요하면 해당 메뉴와 수량을 다시 입력 받습니다.
추가적인 기능으로 각 손님 대해 주문을 따로 관리할 수 있도록 구현하고, 일정 금액 이상 결제하는 손님에게는 더 높은 할인율을 제공합니다. (예시: 5만원 이상 결제한 손님은 20% 할인, 3만원 이상 결제한 손님은 10% 할인, 그 이하의 손님은 기본 5% 할인 등)
여러 손님을 관리하기 위해 각 손님에 대해 개별적으로 주문을 받고, 손님을 응대하는 태도에 대해 사용자에게 선택지를 제공합니다. 사용자의 선택에 따라 매출에 변화가 있도록 구현합니다.
또한 시뮬레이션의 느낌이 나게 하기 위해 '손님 등장, 사장님 등장' 등의 다양한 상황 요소를 구현하고, 하루 단위씩 진행하게 하여 각 날의 매출을 출력합니다.

<br>


## 3. 예상 기능 및 메뉴 (최소 5개)

1. 메뉴 선택
2. 메뉴 수량 입력
3. 추가 주문 여부 질문
4. 손님 응대 태도 선택지 구현
5. 주문 가격에 따른 할인율 측정
6. 할인율을 적용한 총 금액 계산
7. 각 날의 매출 출력
8. 시뮬레이션 느낌의 다양한 상황 요소 구현   
9. 주문 종료

# 🚀 [버전별 개발 일지 & AI 협업 기록]

## 🟦 [1차 과제: V1.0] 기초 뼈대 구축 (입출력과 변수)
### **✨V1.0 개발 내용:**
  - 사용자에게 메뉴판을 볼지 물어보는 질문을 출력했습니다.
  - 메뉴판을 출력한 후 사용자로부터 주문할 메뉴의 수량을 입력 받고 가격을 계산하게 했습니다.
  - 사칙연산을 활용해 기본 할인율 5%를 적용한 총 금액을 출력했습니다.
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:** "C언어 4단계 과제 로드맵(입출력 ➔ 조건문 ➔ 반복문 ➔ 배열/함수)에 적합한 이자카야 아르바이트 시뮬레이션 시스템 시나리오 및 단계별 기능 확장 로드맵 기획 요청"
    * **적용 내용:** '메뉴판 및 총 결제 금액 단순 입출력(1차) ➔ 사용자의 선택에 따른 결과 분기(2차) ➔ 하루 동안 여러 손님 응대 등의 반복 구조 만들기(3차) ➔ 다양한 상황 요소 추가 및 메뉴 관리(4차)'로 이어지는 논리적인 시스템 확장 로드맵과 예상 메뉴를 확립함.
      
 2. **내용 2**
    * **프롬프트 요약:** "1차 과제 필수 요건(변수 5개, 자료형 3개 이상)을 충족하기 위한 손님 응대 구조 모델링 논의"
    * **적용 내용:** 메뉴 선택과 수량 입력에 정수형을 사용하고, 사용자의 대답이 필요할 때에 문자형을 사용하고, 할인율에 대해 실수형을 사용하는 등 자료형을 명확히 분리하였음.
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** 실수형 데이터(할인율이 적용된 총액 계산)를 출력할 때 소수점이 너무 길게 나옴.
     * **해결:** AI 검색을 통해 `printf("%.2f원", total_price_with_discount);`와 같이 서식 지정자를 수정하여 소수점 둘째 자리까지만 출력되도록 개선함.
  2. **문제 2:** 메뉴 가격의 총액을 계산하여 출력할 때 값이 이상하게 나옴.
     * **해결:** AI 검색을 통해 'total_price의 값과 메뉴 수량의 값'은 누적되는 값이기 때문에 변수 선언 시에 0으로 선언해야 한다는 것을 알게 되었고, 이를 적용하여 올바른 계산값이 출력되도록 개선함.
