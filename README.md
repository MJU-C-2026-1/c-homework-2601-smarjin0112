# 💻 [2026-01] 나만의 C언어 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
이자카야 메뉴 주문 및 계산 시스템

<br>


## 2. 시나리오 (5~10줄)
이 프로그램은 이자카야 술집의 아르바이트생이 되었다고 생각하고 메뉴를 입력해 주문을 받고 최종 계산을 하는 시스템입니다.

프로그램이 시작되면 메뉴판을 출력하고, 사용자에게 주문할 메뉴 번호를 입력받습니다. 술과 안주 각각의 수량을 입력 받아, 각 항목의 가격을 계산하고 총액에 반영합니다. 총액 출력 전, 할인율을 적용한 후 할인된 총액을 계산하여 출력합니다.
주문 후에는 사용자가 추가 주문을 할지에 대한 여부를 물어보고, 추가 주문이 필요하면 해당 메뉴와 수량을 다시 입력 받습니다.
또한 추가적인 기능으로 각 테이블에 대해 주문을 따로 관리할 수 있도록 구현하고, 일정 금액 이상 결제하는 손님에게는 더 높은 할인율을 제공합니다. (예시: 5만원 이상 결제한 테이블은 20% 할인, 3만원 이상 결제한 테이블은 10% 할인, 그 이하의 테이블은 5% 할인 등)
여러 테이블을 관리하기 위해 주문을 받기 전 테이블 번호를 입력 받고, 각 테이블에 대해 개별적으로 주문을 받습니다.

<br>


## 3. 예상 기능 및 메뉴 (최소 5개)

1. 메뉴 입력
2. 테이블 번호 입력
3. 술과 안주 수량 입력
4. 추가 주문 여부 질문
5. 주문 가격에 따른 할인율 측정
6. 할인율을 적용한 총 금액 계산
7. 주문 종료

# 🚀 [버전별 개발 일지 & AI 협업 기록]

## 🟦 [1차 과제: V1.0] 기초 뼈대 구축 (입출력과 변수)
### **✨V1.0 개발 내용:**
  - 메뉴판을 출력한 후 사용자로부터 주문할 메뉴 번호를 입력 받고 가격을 계산하게 했습니다.
  - 사칙연산을 활용해 기본 할인율 5%를 적용한 총 금액을 출력했습니다.
  - 사용자에게 추가 주문 여부를 물어보는 질문을 추가했습니다.
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:** "C언어 4단계 과제 로드맵(입출력 ➔ 조건문 ➔ 반복문 ➔ 배열/함수)에 적합한 RPG 캐릭터 성장 시나리오 및 단계별 기능 확장 로드맵 기획 요청"
    * **적용 내용:** 단순한 스탯 계산기를 넘어, '스탯 입력(1차) ➔ 전투력 등급 판별(2차) ➔ 메인 메뉴 시스템(3차) ➔ 다중 캐릭터 관리(4차)'로 이어지는 논리적인 시스템 확장 로드맵과 예상 메뉴를 확립함.
      
 2. **내용 2**
    * **프롬프트 요약:** "1차 과제 필수 요건(변수 5개, 자료형 3개 이상)을 충족하기 위한 캐릭터 초기 스탯 데이터 구조 모델링 논의"
    * **적용 내용:** 포괄적인 의미의 '공격력' 변수 하나를 사용하는 대신, 데이터 특성에 맞춰 문자형(`char` - 이니셜), 정수형(`int` - 체력, 마나), 실수형(`float` - 무기 공격력)으로 자료형을 명확히 분리하여 입력 데이터의 구조적 완성도를 높임.
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** `scanf`로 문자를 입력받을 때 버퍼 문제로 엔터키가 입력되는 현상이 발생함.
     * **해결:** AI에게 질문하여 `scanf(" %c", &initial);` 처럼 `%c` 앞에 공백을 두어 버퍼를 비우는 팁을 얻고 코드를 수정하여 해결함.
  2. **문제 2:** 실수형 데이터(무기 공격력)를 출력할 때 소수점이 너무 길게 나옴.
     * **해결:** 구글링과 AI 검색을 통해 `printf("%.2f", weapon_power);`와 같이 서식 지정자를 수정하여 소수점 둘째 자리까지만 출력되도록 개선함.
  3. **문제 3:** (추가로 발생한 문제와 해결 과정을 객관적으로 자유롭게 기록하세요.)
